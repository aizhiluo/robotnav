13  *
14  * Note that all above 3 concepts have a "type". This type is determined by "nodes" used
15  * to construct the graph. For example, a graph is built to represent the connectivities
16  * among the leaf nodes of a quad tree. Then we have the following three types of graph
17  * elements:
18  * a. Vertex<QuadTreeNode>
19  * b. Edge<Vertex<QuadTreeNode>>
20  * c. Graph<QuadTreeNode>
21  *
22  * A vertex must be associated with a node (such as the tree node); an edge connects two
23  * vertices; a graph is a collection of vertices and the edges of each vertex.
24  *
25  * 2. The graph uses unique IDs to index its vertices. So it's required that the
26  * node used to construct the graph should have an unique ID with the name "node_id_".
27  * Refer to the definition of "struct ExampleNode" defined below.
28  *
29  * 3. A visualized illustration of the graph structure
30  *
31  * Graph "G":
32  * Vertex "V1" - Edge "V1_E1", which connects "V1" to "Vx1"
33  * - Edge "V1_E2", which connects "V1" to "Vx2"
34  * ...
35  * Vertex "V2" - Edge "V2_E1", which connects "V2" to "Vx3"
36  * - Edge "V2_E2", which connects "V2" to "Vx4"
37  * ...
38  * ...
39  *
40  * Vertex "Vx" ...
41  *
42  */
